Workflow Definitions

The structural blueprint defining execution physics, routing logic, and instance lifecycle. This configuration governs how the engine instantiates workflows across IoT, CI/CD, and Automation verticals.

Spec v1.2 Multi-Trigger Enabled State-Aware Persistence

1. Identity & Routing

Core Configuration
Property Type Default Description
definitionId uuid required Immutable unique identifier for this workflow template.
triggers array[obj] [] List of entry points (HTTP, MQTT, Cron). Supports "Logical OR" activation.
correlationKey expression null JSONPath/Expression used to map signals to specific instances (e.g., $.roomId).
onMatch enum CreateNew CreateNew — Ignore existing, start fresh instance.
UpdateExisting — Merge signal into the current instance (IoT/Fan-In).
ReplaceOld — Kill active instance and restart.

2. Execution Physics

Lifecycle & Performance
Property Type Default Description
instanceMode enum Persistent Persistent — Full DB state tracking (Safety/Business).
Ephemeral — In-memory only (Speed/IoT High-Frequency).
executionTimeout duration 0 (infinite) The absolute time limit for a workflow instance before forced termination.
maxActiveInstances int ∞ Concurrency cap to prevent system flooding or resource exhaustion.
priority int (0-10) 5 Execution queue priority. Higher numbers process first in congested environments.
Multi-Trigger Strategy Triggers act as agnostic entry points. Regardless of source (MQTT/HTTP), the correlationKey is resolved against the incoming payload to decide the routing target.

3. Error & Resilience

Global Fault Tolerance
Property Type Default Description
globalErrorHandler nodeId null Fallback node triggered if any unhandled error occurs in the flow.
defaultRetryPolicy object {} Global maxAttempts and backoffStrategy for failing nodes.
isRecoverable bool true If true, engine attempts to resume flow at last checkpoint after server restart.

4. Data & Governance

Context & Security
Property Type Default Description
inputSchema json-schema null Validates trigger payload before instance creation. Rejects invalid data.
secretRefs array[str] [] Keys to fetch from secure Vault storage (API keys, DB credentials).
retentionPeriod duration 30d How long to keep history of completed instances in the audit log.
Versioning Policy By default, active instances stick to the version they were started with. Updates to the definition only apply to new instances created after publication.
